# AUDIT V4-4 — 나의 신앙생활 CSS 분리 1단계

## 목적

V4-4 단계는 기능 변경이 아니라, `style.css`에 섞여 있던 **나의 신앙생활 전용 CSS**를 별도 파일로 분리해 앞으로 수정 범위를 좁히는 단계입니다.

## 변경 파일

- `style.css`
  - my-faith 전용 CSS 블록을 제거했습니다.
  - 공통 토큰과 공통 UI CSS는 그대로 유지했습니다.
- `css/myfaith.css`
  - 기존 `style.css`에 있던 my-faith 전용 CSS를 이동했습니다.
  - CSS 값은 변경하지 않았습니다.
- `index.html`
  - `css/myfaith.css?v=V4-4` 로드 추가
  - 파일 쿼리 버전 V4-4 반영
- `sw.js`
  - `css/myfaith.css?v=V4-4` 캐시 대상 추가
  - 캐시 버전 V4-4 반영
- `sw-update.js`
  - 빌드 버전 V4-4 반영
- `app.js`
  - 동적 파일 호출 버전 V4-4 반영

## 의도적으로 하지 않은 것

- 나의 신앙생활 JavaScript 분리 없음
- 저장/확인/취소 로직 변경 없음
- 지도/길찾기/뒤로가기/위치권한 변경 없음
- `patches.js` 정리 없음
- WebView 파일 반영 없음

## 기대 효과

- `style.css`에서 나의 신앙생활 전용 규칙을 분리하여 CSS 수정 범위를 줄입니다.
- 이후 V4-5에서 나의 신앙생활 JS를 분리하기 전, CSS 의존성을 먼저 안정화합니다.

## 확인 항목

1. 나의 신앙생활 화면 디자인이 V4-3과 동일해야 합니다.
2. 교구·본당 변경/확인/취소 동작이 그대로 유지되어야 합니다.
3. 본당 홈페이지 없는 경우 행 숨김이 유지되어야 합니다.
4. `열기` 버튼 디자인이 그대로 유지되어야 합니다.
5. 새로고침 후 V4-4 CSS가 반영되어야 합니다.
