# AUDIT V4-5 — 나의 신앙생활 JS 분리 1단계

## 목적

V4-5 단계는 기능 변경이 아니라, `app.js` 안에 있던 **나의 신앙생활(PWA) JavaScript 블록**을 독립 파일 `js/myfaith.js`로 분리해 앞으로 수정 범위를 좁히는 단계입니다.

## 변경 파일

- `js/myfaith.js` 새로 추가
- `app.js`에서 나의 신앙생활 IIFE 본문 제거 후 `window.bindMyFaithLifePanel(on)` 호출로 연결
- `index.html`에 `js/myfaith.js?v=V4-5` 로드 추가
- `sw.js` 캐시 목록에 `js/myfaith.js?v=V4-5` 추가
- `sw-update.js` 빌드 버전 V4-5 반영

## 보존한 기능

- 나의 신앙생활 열기/닫기
- 교구·본당 설정/변경
- 임시값/확인/취소 흐름
- 교구와 본당이 모두 있어야 저장되는 조건
- 본당 홈페이지가 없을 때 홈페이지 행 숨김
- 외부 링크 복귀 처리
- 설치 안내 배너와 교구 설정 배너의 표시 조건

## 건드리지 않은 영역

- 지도/길찾기
- 위치권한/현재위치
- 뒤로가기/앱 종료
- 관구·교구 지역검색
- 기도문
- 문의·건의/Firebase
- `patches.js` 정리

## 다음 단계 제안

V4-6에서는 바로 `patches.js` 삭제로 가지 말고, 먼저 **메뉴/주요기능/미사 빠른 사용 팝업 공통화 진단** 또는 **나의 신앙생활 JS 내부 함수 그룹 정리** 중 하나를 선택하는 것이 안전합니다.

추천 순서:

1. V4-6: 나의 신앙생활 JS 내부 섹션 정리 및 함수명 표준화, 기능 변경 없음
2. V4-7: 메뉴/주요기능/미사 빠른 사용 모달 공통 CSS/JS 후보 정리
3. V4-8: `patches.js` 분류표 작성 후 일부 이관

## 확인 항목

1. 나의 신앙생활 버튼이 정상으로 열리는지
2. 교구·본당 변경 화면이 정상으로 열리는지
3. 취소 시 저장되지 않고 이전 단계로 돌아가는지
4. 확인 버튼은 교구와 본당이 모두 있을 때만 저장하는지
5. 외부 링크 열기 후 복귀가 기존처럼 안정적인지
6. 새로고침 후 V4-5 파일이 반영되는지
