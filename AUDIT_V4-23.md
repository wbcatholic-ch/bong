# AUDIT V4-23 — 주요기도문 CSS 분리

## 작업 기준
- 기준 파일: PWA V4-22
- 결과 버전: PWA V4-23
- 목적: 안전한 구조개편 범위 안에서 주요기도문 전용 CSS를 `css/prayer.css`로 분리

## 수행 내용
- `style.css`에 남아 있던 주요기도문 전용 CSS를 새 파일로 이동했습니다.
- 새 파일: `css/prayer.css`
- 디자인 값, 색상, 크기, 레이아웃 값은 변경하지 않았습니다.
- 공통 모듈 헤더/탭 표준 CSS는 아직 `style.css`에 유지했습니다.
- 기도문 JS, 뒤로가기, 빠른메뉴 복귀 로직은 건드리지 않았습니다.

## 분리한 주요 영역
- 주요기도문 헤더 예외 구조
- GoodNews 기도문 버튼 배치/스타일
- 주요기도문 탭 카테고리별 활성 색상
- 주요기도문 화면 기본 레이아웃
- 주요기도문 검색/글자 크기 조절 UI
- 주요기도문 목록 카드
- 주요기도문 상세 본문 카드

## 보류한 항목
- 순례길/웹사이트 JS 정리와 주요기도문 JS 본체 정리는 V4-23에서 함께 진행하지 않았습니다.
- 이유: 현재 분리된 JS 파일이 많고, 기도문/뒤로가기/외부복귀와 연결될 가능성이 있어 한 단계에서 CSS와 JS를 동시에 건드리면 위험도가 높습니다.
- 따라서 V4-24는 별도 단계에서 진행하는 것이 안전합니다.

## 확인 필요 항목
1. 주요기도문 진입 화면이 V4-22와 동일하게 보이는지
2. 주요기도문 탭 색상과 활성 표시가 정상인지
3. 검색창, 글자 크기 버튼, 목록 카드가 정상인지
4. 기도문 본문 보기와 뒤로가기가 정상인지
5. GoodNews 기도문 버튼 위치와 모양이 정상인지
6. 미사 빠른메뉴에서 주요기도문 진입/복귀가 기존과 같은지
7. 커버, 웹사이트, 순례길, 지도/길찾기 화면이 변하지 않았는지

## 다음 추천 단계
- V4-24: 주요기도문 JS 본체 정리 또는 순례길/웹사이트 JS 진단
- 다만 기능 오류가 없다면 먼저 주요기도문 화면 확인 후 진행하는 것이 안전합니다.
