# AUDIT V4-14 — prayer-back.js 분리

## 작업 목적

V4-13 기준으로 `patches.js`에 남아 있던 기도문 뒤로가기/history/빠른메뉴 복귀 전용 컨트롤러를 별도 파일로 분리했다.

이번 단계의 목표는 기능 변경이 아니라, `patches.js`의 역할을 줄이고 기도문 전용 상태 판단을 `js/prayer-back.js`로 독립시키는 것이다.

## 변경 파일

- `js/prayer-back.js` 추가
- `patches.js` 수정
- `index.html` 버전 및 script 로드 수정
- `sw.js` 캐시 목록 및 캐시 버전 수정
- `sw-update.js` 빌드 버전 수정
- `app.js` 동적 파일 호출 버전 수정

## 분리한 내용

`patches.js`에서 아래 로직을 `js/prayer-back.js`로 분리했다.

- 기도문 열림/본문 표시 상태 판단
- 빠른메뉴에서 기도문 진입 여부 판단
- 기도문 본문 → 목록 복귀
- 기도문 목록 → 빠른메뉴 팝업 또는 커버 복귀
- 기도문 복귀 팝업 → 커버 복귀
- 기도문 관련 history trap 호환 함수
- 기도문 복귀 후 pending callback 처리

## patches.js 변화

- V4-13: 약 671줄
- V4-14: 약 424줄

단, `patches.js`의 핵심 popstate/backbutton 컨트롤러는 그대로 유지했다.

아래 흐름은 이번 단계에서 직접 변경하지 않았다.

- 공통 popstate/backbutton 순서
- 커버 뒤로가기 trap 생성
- 지도/길찾기 핵심 함수
- 경로 시트/마커 복원 핵심 흐름
- 위치권한/현재위치
- 나의 신앙생활 저장/취소 로직
- 관구·교구 검색/중복지역 처리
- 새로고침 보호막
- WebView/Android 파일
